# 도서 대출 관리 시스템 - 프로젝트 구조

## 디렉토리 구조

```
library/
├── app/
│   ├── database.py          ← SQLAlchemy 설정, get_db()
│   ├── main.py              ← FastAPI 앱 진입점
│   ├── models/              ← [Data Layer] ORM 모델
│   │   ├── book.py          ← Book (id, title, author, publisher, available)
│   │   ├── member.py        ← Member (id, name, email)
│   │   └── loan.py          ← Loan (id, book_id, member_id, loan_date, return_date, status)
│   ├── repositories/        ← [Data Layer] DB 접근
│   │   ├── book_repository.py
│   │   ├── member_repository.py
│   │   └── loan_repository.py
│   ├── services/            ← [Business Layer] 비즈니스 로직
│   │   ├── book_service.py  ← CRUD, 대출 중 삭제 방지
│   │   ├── member_service.py← 이메일 중복 방지
│   │   └── loan_service.py  ← 대출 가능 여부 확인, 상태 변경
│   ├── routers/             ← [Presentation Layer] Controller
│   │   ├── book_router.py
│   │   ├── member_router.py
│   │   └── loan_router.py
│   └── templates/           ← [View] Jinja2 HTML
│       ├── base.html
│       ├── books/
│       │   ├── list.html
│       │   └── form.html
│       ├── members/
│       │   ├── list.html
│       │   └── form.html
│       └── loans/
│           ├── list.html
│           └── form.html
└── requirements.txt
```

## 실행 방법

```bash
cd library
source .venv/bin/activate
uvicorn app.main:app --reload
# → http://localhost:8000
```

## 강의 내용 적용 현황

| 강의 요소 | 코드 위치 |
|-----------|-----------|
| 3 Layer Architecture | `models/` → `repositories/` → `services/` → `routers/` |
| MVC | `templates/` (View) + `routers/` (Controller) + `models/` (Model) |
| 캡슐화 / 정보 은닉 | Service가 Repository를 감싸고 Router는 Service만 호출 |
| 응집도 高 / 결합도 低 | 각 계층이 인접 계층만 의존 |
| 비기능 요구사항 | 대출 중 삭제 방지, 중복 이메일 방지, 삭제 재확인 JS |
